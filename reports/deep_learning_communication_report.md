# Deep Learning 相关网络原理、任务适配与通信结合学习汇报

## 一、本次整理内容

这次主要整理 Deep Learning 相关网络的基础内容，并把它们和通信方向中可能遇到的任务联系起来。整体上做三件事：

- 先梳理常见网络的基本原理，包括 FNN、CNN、RNN、LSTM、GRU、GAN、Autoencoder 和 Transformer；
- 再对比它们各自适合的数据类型、任务场景和主要区别；
- 最后结合通信任务，整理它们可能用于信道估计、信号检测、CSI 反馈、调制识别、资源分配、波束成形、端到端物理层和信道建模等方向的方式。

整理时的思路是：不只记住每个网络叫什么，而是尽量看清楚它解决的是哪类问题。一个任务到底是固定维度映射、局部模式识别、序列预测、压缩重建、生成建模，还是复杂优化问题，会直接影响应该选择哪类网络。

## 二、八类常见深度学习网络概览

本次主要梳理八类常见网络：FNN、CNN、RNN、LSTM、GRU、GAN、Autoencoder 和 Transformer。

| 模型 | 核心思想 | 适合任务 | 主要局限 |
| :--- | :--- | :--- | :--- |
| FNN / MLP | 信息从输入层单向传到输出层 | 分类、回归、固定维度特征映射 | 不主动利用空间结构或时间顺序 |
| CNN | 用卷积核提取局部特征，并共享权重 | 图像、频谱图、CSI 矩阵、IQ 局部模式 | 更偏局部特征，不天然建模长距离依赖 |
| RNN | 用隐藏状态传递历史信息 | 文本、语音、时间序列 | 长序列中容易梯度消失或梯度爆炸 |
| LSTM | 用门控机制控制记忆保留和遗忘 | 长序列预测、时变信道建模 | 参数较多，训练和推理成本更高 |
| GRU | 简化版门控循环结构 | 中等长度序列、轻量级时序预测 | 表达能力有时略弱于 LSTM |
| GAN | 生成器和判别器对抗训练 | 图像生成、样本生成、数据增强 | 训练不稳定，生成结果需要验证 |
| Autoencoder | 编码器压缩输入，解码器重建输入 | 降维、去噪、压缩反馈、重建 | 可能只学会重构，而没有学到有意义表示 |
| Transformer | 自注意力机制建模全局依赖 | NLP、大模型、多模态、长序列建模 | 计算和显存开销较大 |

这八类模型不是完全同一层级的概念。LSTM 和 GRU 都是 RNN 的改进结构；GAN 是一种生成建模框架；Autoencoder 是一种压缩与表示学习框架；Transformer 则是以注意力机制为核心的序列建模结构。

## 三、各类网络原理、适用任务与区别

### 1. FNN / MLP：前馈神经网络

FNN（Feedforward Neural Network）是最基础的神经网络结构。它由输入层、隐藏层和输出层组成，信息只从前往后传播，不会从后面的层反馈到前面的层。

一层前馈网络可以写成：

$$
h = \sigma(Wx + b)
$$

其中 $x$ 是输入，$W$ 是权重矩阵，$b$ 是偏置，$\sigma$ 是激活函数，例如 ReLU、Sigmoid、Tanh 等。多层前馈网络通常也叫 MLP（Multi-Layer Perceptron）。

FNN 适合处理已经整理好的固定维度特征。例如输入为：

$$
[\text{身高}, \text{体重}, \text{年龄}, \text{血压}, \text{心率}]
$$

这类输入本身是一个向量，各维度含义比较明确，没有明显的空间相邻关系，也没有时间顺序关系。FNN 可以直接学习这些特征与输出结果之间的映射。

FNN 的优点是结构简单、容易理解、实现方便，适合分类、回归、简单决策和特征融合。它也经常作为其他复杂模型的最后几层，把 CNN、RNN 或 Transformer 提取到的特征映射成最终输出。

FNN 的局限是它不会主动利用数据结构。如果把图像直接拉平成一维向量，FNN 并不知道哪些像素是相邻的；如果把一句话直接拼成向量，FNN 也不天然知道词语顺序。因此当数据具有明显空间结构或时间结构时，CNN、RNN 或 Transformer 往往更合适。

### 2. CNN：卷积神经网络

CNN（Convolutional Neural Network）主要用于处理具有局部结构的数据。它的核心是卷积操作：用一个较小的卷积核在输入数据上滑动，并对局部区域做加权求和，从而提取局部特征。

以图像为例，可以先定义一个 $3 \times 3$ 的卷积核，让它在整张图像上滑动。每滑到一个局部区域，就和该区域做一次加权求和，得到一个数。所有位置计算完成后，就得到一张新的特征图。

这种机制有两个重要特点：

- 局部连接：每个卷积核只关注局部区域；
- 权值共享：同一个卷积核在整张图像上复用。

权值共享可以显著减少参数数量。比如一个卷积核在图像左上角、中心、右下角使用的是同一组参数，它只是不断滑动到不同位置检测相同类型的局部模式。

CNN 适合：

- 图像分类；
- 目标检测；
- 语义分割；
- 医学影像和遥感图像分析；
- 频谱图、时频图、CSI 矩阵等具有局部结构的数据。

CNN 和 FNN 的主要区别是：CNN 把“局部结构重要”这个先验写进了模型。FNN 更通用，但直接处理图像或矩阵数据时参数量大，也更难稳定学到局部特征。

### 3. RNN：循环神经网络

RNN（Recurrent Neural Network）用于处理序列数据。它的核心是隐藏状态 $h_t$，当前时刻的状态不仅依赖当前输入 $x_t$，也依赖上一时刻的隐藏状态 $h_{t-1}$：

$$
h_t = f(x_t, h_{t-1})
$$

可以把 RNN 理解成“带记忆的神经网络”。模型在处理当前位置时，会带着前面信息的摘要继续往后处理。

例如一句话：

$$
\text{我 今天 很 开心}
$$

RNN 可以按顺序读取：

- 读到“我”时，结合初始状态得到 $h_1$；
- 读到“今天”时，结合 $h_1$ 得到 $h_2$；
- 读到“很”时，结合 $h_2$ 得到 $h_3$；
- 读到“开心”时，结合 $h_3$ 得到 $h_4$。

因此 RNN 适合文本、语音、时间序列预测等任务。

RNN 的问题是长序列中信息容易衰减。早期信息需要经过很多时间步才能传到后面，每传一步都会被重新压缩一次。训练时误差也要沿时间反向传播，如果梯度不断变小，就会出现梯度消失；如果梯度不断变大，就会出现梯度爆炸。

### 4. LSTM：长短期记忆网络

LSTM（Long Short-Term Memory）是 RNN 的改进版本。普通 RNN 只有隐藏状态，记忆能力有限；LSTM 引入细胞状态和门控机制，用来控制信息保留、写入和输出。

LSTM 中常见的门包括：

- 遗忘门：决定上一时刻的信息保留多少；
- 输入门：决定当前输入写入多少；
- 输出门：决定当前状态输出多少。

这种结构使 LSTM 更适合处理长期依赖。例如一句话中前面的主语可能隔了很多词才影响后面的谓语，普通 RNN 可能已经忘掉前面的信息，而 LSTM 可以通过门控结构保留更久。

LSTM 适合：

- 长文本序列建模；
- 语音识别；
- 时间序列预测；
- 机器翻译；
- 时变信道预测。

LSTM 与 RNN 的区别是：RNN 的记忆传递较粗糙，而 LSTM 用门控机制更精细地控制信息流。代价是 LSTM 参数更多，计算复杂度更高。

### 5. GRU：门控循环单元

GRU（Gated Recurrent Unit）也是 RNN 的改进结构，可以理解为 LSTM 的简化版本。它通常包含更新门和重置门：

- 更新门：决定过去信息保留多少；
- 重置门：决定当前输入和历史状态如何结合。

GRU 的目标和 LSTM 类似，都是缓解普通 RNN 难以捕捉长期依赖的问题。它的结构比 LSTM 更简单，参数更少，训练速度通常更快。

GRU 适合：

- 中短序列预测；
- 文本分类；
- 时间序列分析；
- 算力受限场景下的轻量级序列建模。

GRU 与 LSTM 的区别是：LSTM 结构更细致，表达能力较强；GRU 更轻量，速度更快。在一些任务中 GRU 的效果可以接近 LSTM，但在特别复杂或特别长的序列任务中可能略弱。

### 6. GAN：生成对抗网络

GAN（Generative Adversarial Network）是一种生成模型，包含两个网络：

- Generator：生成器，负责生成样本；
- Discriminator：判别器，负责判断样本是真实样本还是生成样本。

训练过程中，生成器希望生成越来越真实的样本，判别器希望越来越准确地区分真假。两者不断对抗，最终希望生成器学到真实数据的分布。

GAN 适合：

- 图像生成；
- 图像修复；
- 风格迁移；
- 超分辨率；
- 数据增强；
- 稀缺样本生成。

GAN 和 FNN、CNN、RNN 这类分类或预测模型不同，它的目标不是判断类别，而是学习数据分布并生成新样本。GAN 的优点是生成样本通常比较逼真，缺点是训练不稳定，容易出现模式崩塌，即生成器只会生成少数几类样本。

### 7. Autoencoder：自编码器

Autoencoder 是一种压缩与重建结构，由编码器和解码器组成：

$$
z = Encoder(x), \quad \hat{x} = Decoder(z)
$$

编码器把原始输入 $x$ 压缩成潜在表示 $z$，解码器再从 $z$ 重建原始输入 $\hat{x}$。训练目标是让重建结果尽量接近原始输入。

Autoencoder 适合：

- 数据降维；
- 特征提取；
- 图像去噪；
- 异常检测；
- 无监督表示学习；
- 压缩反馈与重建任务。

Autoencoder 的优点是可以在没有标签的情况下学习数据表示。它的局限是，如果模型容量太强，可能只是记住输入，而没有学到有意义的低维特征；如果数据分布太复杂，简单 Autoencoder 的重建质量可能有限。

### 8. Transformer：自注意力网络

Transformer 的核心是自注意力机制。它不再像 RNN 那样按顺序一步步处理序列，而是让序列中的每个位置都可以直接关注其他位置。

注意力机制通常写成：

$$
\text{Attention}(Q,K,V)=\text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V
$$

直观理解是：

- Query：当前位置想找什么信息；
- Key：其他位置提供的索引；
- Value：其他位置真正携带的内容。

每个位置先用 Query 和其他位置的 Key 计算相关性，再按照权重从 Value 中取出信息。

Transformer 的典型模块包括：

- Token Embedding：把输入转成向量；
- Positional Encoding：补充位置信息；
- Multi-Head Self-Attention：从多个角度学习依赖关系；
- Feed Forward Network：对每个位置做非线性变换；
- Residual Connection 和 LayerNorm：稳定深层训练。

Transformer 适合：

- 文本分类、机器翻译、摘要、问答；
- 大语言模型；
- 代码生成与理解；
- 图像分类中的 Vision Transformer；
- 多模态任务；
- 长序列全局依赖建模。

Transformer 与 RNN 的主要区别是：RNN 按时间顺序递归处理，难以并行；Transformer 可以一次性计算所有位置之间的关系，更适合大规模训练和长距离依赖建模。它与 CNN 的区别是：CNN 更偏局部特征提取，Transformer 更偏全局关系建模。

## 四、模型选择思路

选择网络时，可以先问三个问题：

1. 数据是什么结构？
2. 任务输出是什么？
3. 模型需要捕捉局部空间关系、时间顺序关系，还是全局依赖关系？

如果输入只是固定维度特征，FNN 往往就够用；如果数据具有局部结构，可以考虑 CNN；如果数据随时间变化，可以考虑 RNN、LSTM、GRU 或 Transformer；如果任务目标是压缩与重建，可以考虑 Autoencoder；如果任务目标是生成样本，可以考虑 GAN。

| 数据或任务特点 | 更适合的模型 |
| :--- | :--- |
| 固定维度特征映射 | FNN / MLP |
| 图像、频谱图、时频图、CSI 矩阵 | CNN |
| 短序列建模 | RNN、GRU |
| 长序列和长期依赖 | LSTM、Transformer |
| 压缩、降维、重建 | Autoencoder |
| 样本生成、数据增强 | GAN |
| 全局依赖、多模态、大规模序列 | Transformer |

## 五、与通信任务的结合方式

通信中的很多问题也可以转化成深度学习中的常见任务。例如信道估计和信号检测可以看成输入输出映射；CSI 反馈可以看成压缩与重建；调制识别可以看成局部模式识别；信道预测可以看成序列预测；信道建模可以看成分布学习与样本生成。

| 通信任务 | 可结合模型 | 结合思路 | 代表论文 |
| :--- | :--- | :--- | :--- |
| 信道估计与信号检测 | FNN / DNN、模型驱动网络 | 将接收信号、导频信息映射到发送符号或信道状态 | [Ye, Li & Juang, 2017/2018](https://arxiv.org/abs/1708.08514)；[Samuel, Diskin & Wiesel, 2017](https://arxiv.org/abs/1706.01151) |
| CSI 反馈压缩 | Autoencoder、CNN、LSTM / GRU | 用户端压缩高维 CSI，基站端重建 CSI；CNN 提取空间结构，LSTM 利用时间相关性 | [CsiNet, 2017/2018](https://arxiv.org/abs/1712.08919)；[CsiNet-LSTM, 2018](https://arxiv.org/abs/1807.11673) |
| 调制识别与频谱感知 | CNN、RNN、Transformer | 从 IQ 信号或时频图中学习局部模式、时序变化或全局依赖 | [O'Shea, Corgan & Clancy, 2016](https://arxiv.org/abs/1602.04105) |
| 资源分配与波束成形 | FNN / DNN、模型驱动网络 | 学习传统优化算法的输入输出关系，减少在线迭代计算 | [Sun et al., 2017](https://arxiv.org/abs/1705.09412)；[Li & Alkhateeb, 2019](https://arxiv.org/abs/1905.13212) |
| 端到端物理层 | Autoencoder | 把发射机、信道和接收机看成编码、扰动和解码结构，尝试联合优化 | [O'Shea & Hoydis, 2017](https://arxiv.org/abs/1702.00832) |
| 信道建模与数据增强 | GAN | 学习真实或仿真信道分布，生成新的信道样本 | [Orekondy, Behboodi & Soriaga, 2022](https://arxiv.org/abs/2203.08588) |

## 六、几个重点通信结合方向

### 1. 信道估计与信号检测

传统 OFDM 接收机通常先利用导频估计信道，再根据估计出来的信道进行均衡和符号检测。用深度学习理解时，可以把它看成：

$$
\text{接收信号 + 导频信息} \rightarrow \text{发送符号或信道状态}
$$

Ye、Li 和 Juang 的论文 *Power of Deep Learning for Channel Estimation and Signal Detection in OFDM Systems* 就是这个方向的代表。DNN 不一定显式输出 CSI，而是尝试从接收信号中隐式学习信道影响，并恢复发送符号。

这种方法的优势是，在导频较少、没有循环前缀、存在非线性削波噪声等非理想条件下，神经网络可能从数据中学到传统模型难以准确描述的失真规律。问题是模型是否能泛化到新的信道环境。

MIMO 检测中也有类似思路。Deep MIMO Detection 这类工作不是完全黑盒，而是把传统检测算法的迭代结构展开成神经网络层。这属于模型驱动深度学习：保留通信算法结构，再用神经网络学习其中的参数或非线性部分。

### 2. CSI 反馈压缩

Massive MIMO 中，基站需要 CSI 来做预编码。但在 FDD 系统中，下行 CSI 通常要由用户反馈给基站。天线数量越多，CSI 维度越高，反馈开销越大。

这个问题和 Autoencoder 很像：

- 用户端相当于编码器，把高维 CSI 压缩成低维码字；
- 基站端相当于解码器，根据低维码字恢复 CSI。

可以写成：

$$
CSI \rightarrow \text{压缩码字} \rightarrow \widehat{CSI}
$$

CsiNet 就是这个方向的经典工作。它使用 Autoencoder 的压缩重建思想，并用 CNN 提取 CSI 矩阵或角度-延迟域 CSI 中的局部结构。

如果考虑时变信道，连续时刻的 CSI 之间存在时间相关性。这时可以加入 LSTM 或 GRU，让模型利用历史 CSI 辅助恢复当前 CSI。CsiNet-LSTM 就体现了这个思路。

### 3. 调制识别与频谱感知

调制识别可以理解成：

$$
\text{IQ 信号片段} \rightarrow \text{调制类型}
$$

IQ 信号虽然不是图像，但 I/Q 两路可以看成两个通道。不同调制方式在局部波形、幅度变化和相位变化中会表现出不同模式。CNN 的卷积核可以在时间维度上滑动，学习这些局部模式。

O'Shea、Corgan 和 Clancy 的调制识别工作就是使用 CNN 直接处理原始 IQ 信号。这说明 CNN 的本质不是只能处理图像，而是适合具有局部结构的数据。

频谱感知、干扰识别、异常信号检测也可以用类似思路处理。如果输入是频谱图或时频图，可以更直接地使用 CNN；如果需要建模连续时间变化，可以考虑 RNN、LSTM、GRU 或 Transformer。

### 4. 资源分配、波束成形与预编码

通信系统中很多任务本质上是优化问题，例如功率分配、用户调度、波束成形、混合预编码等。传统方法通常需要迭代求解，例如 WMMSE 算法。

深度学习的一种用法是学习传统优化算法的输入输出关系：

$$
\text{信道状态} \rightarrow \text{功率分配 / 波束参数}
$$

Sun 等人的 *Learning to Optimize* 就体现了这个思路。可以先用传统算法生成训练样本，再训练 DNN 近似这个复杂映射。在线运行时，DNN 只需要一次前向传播，速度可能比完整迭代算法更快。

但这类方法必须关注约束问题。通信优化通常不只是输出一个结果，还要满足功率约束、QoS 约束、硬件结构限制等。如果神经网络输出不满足约束，就需要额外的归一化、投影层、惩罚项或模型驱动结构来保证可用性。

毫米波 Massive MIMO 混合预编码中，也可以用深度学习从少量训练导频直接预测预编码矩阵，从而减少传统信道估计和波束搜索开销。

### 5. 端到端物理层设计

Autoencoder 还可以用于理解端到端物理层。传统通信系统通常把调制、编码、信道估计、均衡、检测、译码拆成多个模块分别设计；端到端学习则尝试把整条链路看成一个整体。

可以类比为：

| Autoencoder | 通信系统 |
| :--- | :--- |
| 输入数据 | 待发送消息 |
| 编码器 | 发射机 / 调制编码模块 |
| 中间表示 | 发射信号 |
| 噪声扰动 | 无线信道 |
| 解码器 | 接收机 / 检测译码模块 |
| 重建输出 | 恢复消息 |

O'Shea 和 Hoydis 的物理层深度学习论文提出了这种思路。它的意义是提供一种新的建模角度：发射机和接收机不一定只能分开设计，也可以尝试联合优化。

不过真实系统还要考虑功率、频谱、同步、硬件、信道变化和标准协议。因此我认为端到端物理层学习更适合作为理解框架和研究方向，而不是简单替代现有通信系统。

### 6. 信道建模与数据增强

GAN 可以用于信道样本生成和数据增强。通信仿真需要大量信道样本，但真实测量成本高，传统统计信道模型又可能难以覆盖复杂场景。GAN 可以尝试学习真实或仿真信道数据的分布，再生成新的信道样本。

训练时：

- 生成器生成“像真实信道一样”的样本；
- 判别器判断样本是真实信道还是生成信道；
- 两者对抗训练，使生成器逐渐学习信道分布。

MIMO-GAN 就是用 GAN 做 MIMO 信道生成建模的代表工作。

但这里要谨慎：生成信道在统计特征上相似，不一定说明它对所有通信算法评估都可靠。如果要用于系统仿真，还需要验证生成样本是否保留关键物理特性。

### 7. Transformer 与通信任务

Transformer 可以用于通信中需要全局关系建模的任务。例如：

- 长序列信道预测；
- 多天线之间的关系建模；
- 多子载波之间的相关性建模；
- 多模态通信感知；
- 通信网络流量预测。

它的优势是可以同时关注多个时间点、多个天线或多个子载波之间的关系，而不需要像 RNN 那样一步步递推。缺点是计算和存储开销较大，因此在 UE 或边缘设备上部署时需要考虑复杂度和轻量化。

## 七、目前的阶段性理解

我目前的理解是，深度学习在通信中更像传统通信理论的补充工具，而不是替代品。通信理论提供了信道、噪声、调制、编码、功率约束、QoS 约束和优化目标等基本结构；深度学习可以在以下场景发挥作用：

- 信道或硬件失真复杂，难以建立精确模型；
- 传统优化算法效果好，但在线迭代计算复杂；
- 通信数据本身具有明显空间结构或时间结构；
- 某些任务天然接近压缩、重建、预测或生成。

对我们组做算法的方向来说，比较值得关注的是“模型驱动 + 数据驱动”的结合。也就是说，不完全依赖黑盒网络，而是在保留通信算法结构和物理约束的基础上，用神经网络学习难以建模或难以手工调参的部分。

例如：

- 将传统迭代检测算法展开成神经网络层；
- 用 DNN 近似 WMMSE 等复杂优化算法；
- 在 Autoencoder 中加入通信系统约束；
- 在 CSI 反馈中结合 CNN 的空间特征提取和 LSTM 的时序建模。

## 八、后续学习方向

后续如果继续深入，我认为可以优先看三个方向：

1. 资源分配和波束成形中的深度学习方法
   重点理解神经网络如何处理功率约束、QoS 约束、泛化能力和复杂度问题。

2. CSI 反馈与 CsiNet 系列方法
   重点理解 Autoencoder、CNN、LSTM 如何组合，以及压缩率、重建误差和系统性能之间的关系。

3. 模型驱动深度学习
   重点理解传统通信算法如何展开成网络结构，以及这种方式相比纯黑盒 DNN 有什么优势。

这次学习主要完成了对常见深度学习网络的整体梳理，以及它们与通信任务之间的初步对应。后续还需要结合具体论文、仿真代码和实验结果，进一步理解这些方法在真实通信场景中的性能、复杂度和适用边界。
