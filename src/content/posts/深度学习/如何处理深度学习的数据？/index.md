本文参考知乎博主[咖啡因是恶魔](https://www.zhihu.com/people/dong-ye-zi-icycoconut)的教程[《我该干啥》之新手如何入门深度学习](https://zhuanlan.zhihu.com/p/27337809985)，整理深度学习中“数据处理”这一部分的基础概念。

相比一些纯概念内容，数据处理更偏工程实践，刚开始学习时容易觉得零散。

一个基础理解是：很多深度学习任务本质上都可以看成训练一个函数 $g(x)$，让它根据输入 $x$ 给出尽可能接近真实答案的预测结果。

那么接下来的问题就变成了：

```text
x 从哪里来？
y 从哪里来？
它们在计算机里到底长什么样？
怎样才能把它们送入模型？
```

因此，数据处理并不是在讲新的模型，而是在讲深度学习项目开始前的重要准备工作：把现实里的数据整理成机器能计算的数字形式。

本文仅表达个人见解，可能会有错误，欢迎指正交流。:)

# 数据处理在整个流程里的位置

深度学习中常见的预测关系可以写成：

$$
g(x) \rightarrow \hat{y}
$$

其中 $x$ 是输入，$\hat{y}$ 是模型预测出来的结果。训练时还需要真实答案 $y$，用来判断 $\hat{y}$ 和 $y$ 差多少。

但这里有一个很实际的问题：现实里的图片、文字、声音、表格、轨迹数据，并不是天然就能送进模型的。模型真正能处理的是数字，而且通常是按固定结构组织好的数字。

因此数据处理其实是在补上这一步：

```text
现实数据
→ 读取
→ 检查和清洗
→ 转成 tensor
→ 组织成 X 和 Y
→ 一批一批送进模型
```

也就是说，在正式训练模型之前，必须先把函数的输入和答案准备好。

# 不要只说数据叫什么

描述数据时，常用“名字”来概括数据类型，比如：

- 这是图片数据
- 这是天气数据
- 这是 GPS 轨迹数据
- 这是通信信号数据

但这种说法其实对程序没有太大帮助，它们难以被机器理解，而且同样叫“天气数据”，可能有完全不同的格式。

比如：

```text
只记录每天温度：
365 个数字

记录每天温度、湿度、气压：
365 天 × 3 个指标

每小时记录一次，连续 30 天：
720 个时间点 × 3 个指标
```

它们都可以叫天气数据，但对于模型来说，它们完全不是同一种形状。

因此描述数据时，应该尽量从这些角度说清楚：

| 问题 | 例子 |
| :--- | :--- |
| 有多少条样本 | 1000 张图片、500 条轨迹、20000 行表格 |
| 每条样本长什么样 | 一张 RGB 图片、一段长度为 100 的序列 |
| 每条样本包含多少数字 | `3 × 128 × 128`、`100 × 5` |
| 每个数字是什么类型 | 整数、浮点数、布尔值 |
| 数值范围是多少 | `0~255`、`0~1`、任意实数 |
| 数据之间有什么关系 | 图像相邻像素相关，时间序列前后相关 |
| 标签是什么 | 类别编号、连续数值、每个像素的类别 |

这里最重要的思维变化是：不要先执着于“这个数据叫什么”，而是先问“这个数据在计算机里是什么形状”。

# Tensor：模型真正看到的数据

PyTorch 里最核心的数据容器是 `tensor`。目前可以先把它理解成多维数组。

不同维度的 tensor 大概可以这样看：

| 形式 | 例子 | 理解方式 |
| :--- | :--- | :--- |
| 0 维 tensor | `3.14` | 一个数字 |
| 1 维 tensor | `[1, 2, 3]` | 一串数字，也就是向量 |
| 2 维 tensor | `[[1, 2], [3, 4]]` | 表格或矩阵 |
| 3 维 tensor | `(3, 128, 128)` | 一张 RGB 图片 |
| 4 维 tensor | `(32, 3, 128, 128)` | 一批 RGB 图片 |

重点不是想象高维空间，而是能读懂它的 `shape`。比如：

```text
(32, 3, 128, 128)
```

可以被理解为：

```text
一批 32 张图片；
每张图片有 3 个颜色通道；
每个通道高 128、宽 128。
```

这里的第一个维度通常是 batch，也就是一批样本的数量。

# 一个 tensor 不一定只是一条样本

一个 tensor 通常可以看成用一组数据描述的一个样本，这个理解在很多情况下是对的，但还需要补充一点。

一个 tensor 既可以表示一条样本，也可以表示一批样本。

例如：

```text
一张彩色图片：
shape = (3, 128, 128)

一批 32 张彩色图片：
shape = (32, 3, 128, 128)
```

所以更准确地说：

```text
一条样本可以用 tensor 表示；
一批样本也可以合并成一个更高维的 tensor。
```

这也是为什么训练代码里经常看到 `batch_size`。模型一般不是一次只看一条数据，而是一批一批看。

# X 和 Y 的格式

监督学习里不只需要输入 $X$，还需要标签 $Y$。

比如图像分类任务：

```text
输入 X：图片
标签 Y：图片属于哪一类
```

如果一批数据有 32 张 RGB 图片，每张图片大小是 `128 × 128`，那么：

```text
X.shape = (32, 3, 128, 128)
Y.shape = (32,)
```

其中 `Y` 可能是：

```text
[0, 1, 1, 3, 2, ...]
```

每个数字代表一个类别编号。

如果是回归任务，比如根据房屋信息预测房价：

```text
X.shape = (100, 8)
Y.shape = (100, 1)
```

这里可以理解成：一批有 100 套房，每套房有 8 个特征，标签是每套房对应的价格。

因此分析一个任务时，可以先问：

```text
输入 X 是什么？
X 的 shape 是多少？
标签 Y 是什么？
Y 的 shape 是多少？
```

这比直接讨论“用什么模型”更基础。

# 数据清洗：先别急着训练

数据处理里还有一部分是数据清洗、统计和可视化。这部分看起来很工程化，但实际非常重要。

因为模型会认真学习输入给它的数据。如果数据里本身有错误，模型不会自动知道那是错误。

比如：

```text
身高 = -170
温度 = 65535
经纬度 = NaN
房价 = 无穷大
图片文件打不开
标签写错了
同一条数据重复了很多次
```

这些问题如果不提前处理，后面训练时可能会出现异常现象，比如 loss 变成 `nan`，或者模型训练效果很差但难以定位原因。

所以在真正训练前，应该先检查：

- 数据量够不够
- 有没有缺失值
- 有没有异常值
- 有没有重复数据
- 图片尺寸是否一致
- 标签有没有明显错误
- 各个类别数量是否严重不平衡
- 数值范围是否需要标准化

数据标准化可以理解为：把数据缩放到模型更容易处理的范围。

比如图像像素原本常见范围是：

```text
0 ~ 255
```

可以除以 `255` 变成：

```text
0 ~ 1
```

这样模型训练时会更稳定一些。

# 可视化不是为了好看

数据处理中还需要重视可视化。这里的可视化不是为了让图表好看，而是为了快速发现数据里的问题。

比如：

- 画类别数量柱状图，看类别是否平衡
- 画数值分布直方图，看是否有异常值
- 随机展示几张图片，看图片和标签是否对应
- 画轨迹图，看 GPS 轨迹是否异常
- 画时间序列曲线，看数据是否有突然跳变

数据可视化相当于在训练前先对数据质量做抽查。如果数据本身就很混乱，模型很难学到稳定规律。

# Dataset：管理单条数据

PyTorch 里自定义数据集通常会继承 `Dataset`。可以把 `Dataset` 理解为样本管理器，它至少要回答三个问题：

```python
class MyDataset(Dataset):
    def __init__(self):
        pass

    def __len__(self):
        pass

    def __getitem__(self, idx):
        pass
```

这三个函数分别对应：

| 方法 | 职责 |
| :--- | :--- |
| `__init__` | 准备样本列表，比如保存路径、读取文件名、配置预处理方式 |
| `__len__` | 告诉程序数据集里一共有多少条数据 |
| `__getitem__` | 根据编号取出某一条数据；在监督学习中通常返回 `(x, y)` |

例如：

```text
dataset[0] -> 第 0 张图片和它的标签
dataset[1] -> 第 1 张图片和它的标签
```

所以 `Dataset` 主要解决的是“如何取一条数据”的问题。

# DataLoader：一批一批送数据

训练时模型通常不是一条一条看数据，而是一批一批看数据。

所以 `DataLoader` 可以理解成：

```text
DataLoader = 批量数据迭代器
```

它会从 `Dataset` 里取出多条样本，打包成一个 batch。

例如：

```python
loader = DataLoader(dataset, batch_size=32, shuffle=True)
```

这表示：

```text
每次取 32 条数据；
每个 epoch 前重新打乱顺序。
```

二者的关系可以先记成：

```text
Dataset 管单条数据；
DataLoader 按 batch 迭代数据。
```

模型真正训练时拿到的通常是：

```text
X_batch, Y_batch
```

而不是单独的一条 `x, y`。

# 数据增强：提升模型泛化能力

数据增强可以理解成：在不改变标签含义的前提下，对原始数据做一些合理变化。

比如一张苹果图片：

- 轻微旋转
- 调整亮度
- 稍微裁剪
- 缩放
- 加一点模糊

它大多数情况下还是苹果。

这样做的目的，是让模型不要只记住某一张图片里的那个苹果，而是学到更稳定的特征。

但数据增强不能乱来。判断一个增强是否合理时，关键问题是：

```text
增强之后，标签还成立吗？
```

比如行人检测时，把图片上下翻转可能就不合理，因为现实场景中的行人通常不会倒置出现。天气预测里把雨天随机改成晴天也不合理，因为这已经改变了数据本身的含义。

# 训练集、验证集和测试集

数据处理的最后还涉及数据集划分。


| 数据集 | 作用 |
| :--- | :--- |
| 训练集 | 用来学习模型参数 |
| 验证集 | 用来调参、选择模型、观察是否过拟合 |
| 测试集 | 在最后用于评估模型泛化效果 |

不能把所有数据都拿来训练，是因为模型可能只记住训练样本。如果模型只是在训练集上表现好，但遇到没见过的数据就不行，这就是过拟合。

所以验证集和测试集的意义，是检查模型有没有真的学到规律，而不是只记住训练数据。

常见比例可以先用：

```text
训练集 : 验证集 : 测试集 = 8 : 1 : 1
```

当然这不是固定规则，只是初学时比较容易理解和使用。

# 总结

数据处理一开始容易显得零散，是因为它把很多工程细节放在了一起：tensor、shape、数据清洗、数据加载、数据增强、数据集划分。将这些概念串起来之后，主线会更清楚。

这条主线是：

```text
现实数据
→ 描述数据格式
→ 检查和清洗数据
→ 转成 tensor
→ 组织成 X 和 Y
→ Dataset 负责取单条数据
→ DataLoader 负责打包一批数据
→ 送进模型训练
```

因此这一部分并不是在讲模型怎么变聪明，而是在讲：模型开始学习之前，需要先把数据整理成它能看懂、能计算、能一批一批读取的形式。

因此需要重点记忆的是：

- `tensor`：用来装数字的多维数组
- `shape`：数据的形状
- `X`：输入
- `Y`：标签或答案
- `Dataset`：管理单条数据
- `DataLoader`：把数据打包成 batch
- 数据清洗：提前发现脏数据和异常数据
- 数据增强：在合理范围内增加数据多样性

如果用一句话概括数据处理：

> 深度学习不只是找函数 $g(x)$，还要先把现实数据整理成机器能看懂的 $x$ 和 $y$。
